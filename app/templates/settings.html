Primary / StandbyHow two Opsbook installs work today

Current supported flow is primary-to-standby via encrypted Emergency Export and Import. The standby should use the same `OPSBOOK_SECRET_KEY` and `EXPORT_SECRET_KEY`, stay read-only, and be promoted manually if the primary is down.

Bidirectional multi-instance sync is not enabled yet. It needs conflict handling for edited records and encrypted secrets so two writable instances do not quietly overwrite each other.
