Primary / Mirror SetupPlain-English secondary instance steps

Today, Opsbook supports a simple primary plus mirror model. The primary is where you normally write. The mirror imports encrypted backups, stays read-only, and can be promoted if the primary is down.

  1. Install a second Opsbook stack on another machine.
  2. Use the same OPSBOOK_SECRET_KEY and EXPORT_SECRET_KEY as the primary so encrypted data can be read after import.
  3. Set the mirror to INSTANCE_MODE=standby so normal edits are blocked.
  4. Create an Emergency Export on the primary and import it on the mirror.
  5. If the primary is down, change the mirror to INSTANCE_MODE=primary and restart it.

Bidirectional live sync is not enabled yet. That is deliberate: two writable copies can create conflicts, especially for encrypted secrets. The current flow is boring on purpose, which is exactly what you want in an emergency.
